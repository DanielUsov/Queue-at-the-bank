# Очередь в банке

## Описание

Приложение для составления, отображения и администрирования электронной очереди в физическом отделении банка.
Приложение имеет три логических блока:

- Пользовательский - функционал доступный посетителям отделения, дерево блоков при помощи которого определяется тип клиента(цели его прихода в отделение). Интерфейс доступен на терминалах, планшетах сотрудников и смартфонах пользователей по QR-коду.
- Очередь - представляет собой отображение актуальной очереди и вывода на экраны для понимания посетителями своей очередности.
- Административный/служебный - Представление всего пула посетителей с отображением их типов и времени ожидания. Имеется функционал выбора клиента, то-есть приглашения посетителя к себе(к работнику который специализируется на его вопросе).

### Особенности

Приложение исходя из описания имеет несколько явных особенностей:

1. Оно является децентрализованным, то-есть в каждом отделении банка будет собственный экземпляр приложения, следовательно, на приложение не будет оказываться сильной нагрузки.
2. Важна скорость добавление нового посетителя банка в очередь.
3. В пользовательском интерфейсе, особенное в первых двух блока, приоритет отдается UX.
4. Наличие альтернативного языка интерфейса(английского)

#### Техническое описание

Сущности и их возможности:

1. Посетитель - имеет лейбл(клиента #, где # буква алфавита, в случаи превышения ограничения в кол. букв алфавита, после буквы добавляется порядковый номер, пока не будет доступных букв алфавита), Время создания(через которое определяется время ожидания), тип вопроса
2. Менеджер - имеет ФИО, должность. Имеет доступ к актуальной очереди посетителей.
3. Админ - наследует все поля Менеджера, но имеет возможность администрировать аккаунты менеджеров.
